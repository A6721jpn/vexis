# VEXIS-CAEの解説

**AIを使う目的**
ラバードーム（メンブレンキーボード等）の非線形座屈解析を、設計者自身がワンクリックで実行できるよう自動化。

**従来作業のどこを効率化したかったのか**

ラバードームの非線形解析には3つの高いハードルがあった：

1. **O-gridメッシュ生成の困難さ**
   - 座屈解析の収束性を確保するには、ドーム中心部にO-grid（バタフライメッシュ）が必要
   - 商用メッシャー（SIMULIA等）ではこれに高価な追加ライセンスと複雑な手順が必要
   - 結果として、多くの現場では理想的なメッシュを諦めていた

2. **非線形モデルの再定義の複雑さ**
   - 解析モデルのメッシュを差し替えると節点番号が変わり、接触面・境界条件が壊れる
   - これを手動で再設定するには非線形解析の専門知識が必要
   - 解析専任者でなければ対応困難

3. **解析専任者への依存**
   - 設計者が「形状を変えて試したい」と思っても解析担当者に依頼して順番待ち
   - 設計者の発想を素早く検証できず、設計探索の自由度が制限されていた

**使用したAI**
Google Antigravity + Gemini, Google AI proプラン

**従来の作業と比較してどれだけ効率化されたか**
- 解析1ケースあたり1〜2時間 → ワンクリックで自動完了
- 解析専任者への依頼待ち → 設計者自身で即時実行
- 設計検討のイテレーション速度が劇的に向上

**主要コンポーネント**
- `src/mesh_gen/`: O-gridメッシュ生成（商用ライセンス不要）
- `src/mesh_swap/set_reconstructor.py`: 接触・境界条件の自動再構築（KDTree + 幾何ルール）
- `src/libs/waffleiron/`: FEBioの.xpltバイナリを直接パース
- `src/gui/`: ダークテーマのモダンGUI（PySide6）

**その他・気づいた点など**

- **真の価値**: 解析自動化ではなく「設計者のエンパワーメント」。解析専任者への依存を排除し、設計者が自由な発想をすぐに検証できる環境を構築。
- 約2ヶ月の開発期間、200ファイル規模。AI駆動IDE（Antigravity）なしでは現実的でなかった。
- O-gridメッシュ生成とSetReconstructorが核心技術。商用ツールの制約を回避しつつ、専門家の暗黙知を自動化。
- Gitのロールバック手段は必須。AIがコードを壊すことがある。
- 年数回の作業でも自動化価値あり。設計探索の幅が「依頼可能な回数」から「発想できる回数」に変わる。
